# Realm TCG — Bestiary of the Sundered Age

A vanilla-JS 1v1 fantasy card game. Build a 30-card deck from a Bestiary of
original creatures, wield Legendary signature moves, and out-strategize your
rival. Vanilla JS modules in the browser, plain Node on the server, no
mandatory build step. Backed by Supabase for the Bestiary and persistent
accounts. Deploys to Vercel (HTTP routes) with a Dockerfile included for any
persistent-process host (Fly.io / Render / VPS).

> **Original-IP reskin.** This project is a thematic reskin of an upstream
> Pokémon-themed TCG engine into an original fantasy-MMO setting. **No
> Nintendo / Game Freak / The Pokémon Company intellectual property remains in
> the shipped build** — the creature scraper and seed, all official sprite/cry
> CDN URLs, type-chart icons, trainer art, and the in-code "Pokémon" naming
> have been stripped or replaced. Creatures, champions, schools, and copy are
> original. See the IP note under [License](#license).

## Features

- Solo vs AI on Easy / Medium / Hard
- 6 playable **Champions** with passive abilities (Thordak the Stonewarden,
  Lyralei Tidecaller, Zix the Sparkthief, Sylvanis Greenmother, Vael the
  Mindweaver, Drakkonir Wyrmlord)
- Every creature has a free Basic attack plus a school-flavored Special
  (Fire → Inferno + Burn, Storm → Thunderclap + Stun, Martial → Crushing Blow
  + Bleed, etc.)
- 18-school effectiveness chart (martial, fire, tide, storm, verdant, frost,
  brawl, plague, earth, sky, mind, swarm, stone, spectral, wyrm, shadow, iron,
  radiant)
- Status effects: Burn, Bleed, Stun, Freeze, Sleep, Poison, Curse
- Passkey accounts (WebAuthn), Supabase-backed
- Collection viewer + multi-deck builder
- Per-match reward drops, achievements with toast notifications
- Match history, global leaderboard
- "Fight a Champion" boss battles, daily challenges
- Multiplayer matchmaking + private 6-character room codes (with QR
  deep-link auto-join). Multiplayer needs a persistent-process host; see
  `DEPLOY.md`.

## Local development

```bash
yarn install
cp .env.example .env        # fill in your Supabase + SESSION_SECRET
# Bootstrap the DB: run scripts/schema.sql then scripts/schema-accounts.sql
# in the Supabase SQL editor (or `supabase db push` — see DEPLOY.md).
node scripts/seed-bestiary.js   # loads shared/bestiary.json into the bestiary table
yarn start                  # node server.js — listens on :3000
```

Open `http://localhost:3000` (passkeys reject IP-based hosts).

## The Bestiary

`shared/bestiary.json` holds the creature data. The current build ships a
**curated 32-creature sample** spanning all 8 families (Humanoid, Dragon,
Undead, Demon, Beast, Elemental, Aberration, Fey), tiers 1–4, and all 18
schools. It is generated by `scripts/build-bestiary-sample.js` — the seam for
scaling toward the full 1,025-entry roster: add rows, re-run, re-seed.

Each entry carries an `art_prompt` (descriptive late-1990s painted box-art
style). `scripts/generate-art.js` renders those prompts through a configurable
image API (fal.ai / Replicate / OpenAI) into `client/assets/creatures/`. That
script is **manual-run only** — never invoked by CI or the build. Cards show a
neutral "artwork pending" placeholder until art is produced.

## Tests

```bash
yarn test
```

555 `node:test` unit tests cover the school chart, damage formula, ability /
signature system, status effects, sessions, deck-builder boundaries, spells,
and game flow.

## Deploy

See `DEPLOY.md`. TL;DR — `vercel deploy --prod` for HTTP routes; a separate
persistent host (Fly.io with the included Dockerfile) for the Socket.IO
multiplayer server.

## Project status

Core loop, mechanics, schools, champions, achievements, and the curated
Bestiary sample are reskinned and IP-clean. A **deeper content pass** is still
pending for the narrative surfaces that reference individual creatures and
depend on the full roster: story-mode chapters, reading-mode stories, and the
daily boss/puzzle anchors. Until then those modes may read as placeholder.
